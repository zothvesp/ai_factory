# Documentation Loop

## Purpose

The Documentation Loop preserves durable knowledge in a form future agents and
engineers can use without additional explanation.

## Entry Criteria

- Audience, purpose, scope, and source material are identified.
- Relevant standards and existing documents are known.
- Completion rules from the manifest apply.

## Activities

- Read source material before writing.
- Define rules, rationale, examples, decision guidance, AI guidance, review
  checklist, and references.
- Cross-link instead of duplicating established standards.
- Remove placeholders and stale guidance.
- Update manifest, state, changelog, next task, and Project Brain when closing a
  phase.

## Outputs

- Production documentation.
- Updated indexes and cross-links.
- Phase records.
- Project Brain updates.

## Exit Criteria

- Documentation satisfies manifest completion rules.
- No placeholder sections remain in completed scope.
- Future agents can apply the document independently.

## Checklist

- Purpose and scope are clear.
- Rules include rationale and exceptions.
- Examples are actionable.
- Cross-links are useful.
- Phase metadata is current.
